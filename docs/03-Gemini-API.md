# Chapter 3: Gemini API

## Overview

The Gemini API provides developers with access to Google's family of large language models (LLMs), enabling applications to understand natural language, generate responses, extract structured information, summarize content, write code, and perform many other AI-powered tasks.

During this learning journey, I learned how to authenticate with the Gemini API using an API key, send prompts to the model, receive responses, and integrate Gemini with Google Agent Development Kit (ADK).

The Gemini model served as the reasoning engine for the Ambient Expense Agent developed throughout this project.

---

# What is Gemini?

Gemini is Google's family of multimodal Large Language Models (LLMs).

These models are capable of:

- Natural Language Understanding
- Text Generation
- Code Generation
- Summarization
- Information Extraction
- Function Calling
- Structured Output Generation
- Reasoning over complex tasks

Gemini can process different types of inputs including:

- Text
- Images
- Audio
- Video
- Documents

making it suitable for building intelligent AI applications.

---

# Gemini API

The Gemini API allows applications to communicate with Gemini models through simple API calls.

Instead of building an AI model from scratch, developers can send prompts to Gemini and receive intelligent responses.

Typical workflow:

User Input

↓

Gemini API

↓

Gemini Model

↓

Generated Response

---

# Authentication

To use Gemini API, authentication is required.

There are two supported approaches:

## 1. Google AI Studio API Key

This method is suitable for local development.

Environment Variable:

```
GEMINI_API_KEY=YOUR_API_KEY
```

The API key is securely loaded using a `.env` file.

---

## 2. Vertex AI

Google Cloud Vertex AI provides enterprise-grade authentication and deployment.

Environment Variables:

```
GOOGLE_GENAI_USE_VERTEXAI=true

GOOGLE_CLOUD_PROJECT=your-project-id

GOOGLE_CLOUD_LOCATION=global
```

During this learning project, I used the Gemini API with an API key for local testing.

---

# Installing the SDK

Google provides the `google-genai` Python SDK.

Installation:

```bash
pip install google-genai
```

This SDK simplifies interaction with Gemini models.

---

# Creating a Gemini Client

A client object establishes communication with Gemini.

Example:

```python
from google.genai import Client

client = Client(api_key=os.getenv("GEMINI_API_KEY"))
```

The client handles authentication and API communication.

---

# Generating Responses

A response can be generated by providing:

- Model
- Prompt
- Configuration

Example:

```python
response = client.models.generate_content(
    model="gemini-2.5-flash",
    contents="Say hello"
)

print(response.text)
```

Output:

```
Hello!
```

During the project, I successfully tested my Gemini API configuration using this example before integrating it into Google ADK.

---

# Structured Output

One of Gemini's powerful features is generating structured data.

Instead of plain text responses, Gemini can produce JSON-like outputs.

Example:

User:

```
Lunch with client for $85
```

Gemini Output:

```json
{
  "description": "Lunch with client",
  "amount": 85,
  "category": "Food"
}
```

This structured output was used in the Ambient Expense Agent to extract expense information automatically.

---

# Retry Options

The Gemini SDK supports retry mechanisms.

Example:

```python
retry_options=types.HttpRetryOptions(
    attempts=3
)
```

This improves reliability by automatically retrying failed requests.

---

# Integration with Google ADK

Google ADK integrates directly with Gemini.

Example:

```python
model=Gemini(
    model="gemini-2.5-flash",
    retry_options=types.HttpRetryOptions(
        attempts=3
    ),
)
```

The LlmAgent automatically sends prompts to Gemini and returns structured outputs.

This greatly simplifies AI agent development.
